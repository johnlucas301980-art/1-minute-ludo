Phase 4.2 — Flutter Wallet Service

Read the current architecture, API documentation, coding rules, and the newly completed Phase 4.1 backend before writing any code.

### Goal

Implement the Flutter service layer for the Wallet feature using the backend endpoints completed in Phase 4.1.

This phase is **service layer only**.

Do **not** build any Wallet UI.

---

### Use the existing architecture

Follow the same patterns already used by:

* ProfileService
* ChangePasswordService
* AuthService

Reuse:

* ApiClient
* ApiException
* SessionExpiredException
* TokenStorage
* Existing models and testing style

Do not introduce new dependencies or state-management libraries.

---

### Backend endpoints

GET /api/wallet

GET /api/wallet/history

---

### Create

mobile/lib/features/wallet/models/wallet.dart

mobile/lib/features/wallet/models/wallet_transaction.dart

mobile/lib/features/wallet/services/wallet_service.dart

mobile/test/features/wallet/wallet_service_test.dart

### WalletService

Implement:

* Future<Wallet> getWallet()

* Future<WalletHistory> getHistory({
  int limit = 20,
  int offset = 0,
  })

Both methods must use authenticated requests through the existing ApiClient.

Wallet

WalletTransaction

* type

WalletHistory

* transactions

### Error handling

Keep behaviour identical to existing services.

Propagate:

* ApiException
* SessionExpiredException
* Network exceptions

No custom wallet exceptions are needed in this phase.
